# 2. Constraints and Requirements

## 2.1 Technical Constraints

- Must follow **hexagonal architecture**
- Cluster-aware services for high availability
- Secure token issuance for M2M authentication
- Use `did:web` for DID documents
- W3C VC and DCP compliance

## 2.2 Business / Functional Requirements

- Create / manage participant contexts
- Issue / revoke / reissue credentials
- Generate / publish DIDs
- Provide internal management APIs (Identity API)
- Integrate with external Verifiable Data Registries (VDR)
